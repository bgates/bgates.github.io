Day Four has us figuring out when to sneak by a sleeping guard, based on a set of notes about when each guard starts a shift, falls asleep, and wakes up. The data format looks like this:

[1518-11-01 00:00] Guard #10 begins shift
[1518-11-01 00:05] falls asleep
[1518-11-01 00:25] wakes up

We're told all the sleeping / waking events happen between midnight and 1am, so only the minute part matters for those. (The date part of the datetime stamps is in the distant past because of a cute backstory involving time travel.) For part one, we need to find which minutes each guard is asleep (for instance, one guard might always be asleep at 12:15). We also need to find which guard has the most minutes asleep.

Once I had the data sorted (so to speak), I started to think about what kind of JavaScript structure I wanted to use to store it. I'll have data about each of several guards, which means I'll need some kind of container, which in JavaScript makes me think of either object or array (and usually object). Every guard has a numerical id, but the ids aren't zero-indexed and the order of them doesn't seem important, so I will make an object and use the guard ids as keys. The value associated with a key will be the information about a guard. What will that look like? At first, I thought I had to keep track of two pieces of information per guard, so I made a little object:

Then I realized that numberOfMinutesAsleep could be calculated by whichMinutes.length, so I simplified the object to have just one property:

Then I realized having an object just makes it a little harder to get to the array in which I'm keeping track of the minutes, so I ended up just using an array to represent the minutes a guard spends asleep. The array which tracks the minutes a guard spends asleep gets paired with the key which is that guard's id. For example, using the example guard data above (id #10, falls asleep at 00:05, wakes up at 00:25), I could write:

OK, I have an array of strings in data, and I want to end up with an object. My initial inclination is to do that with.reduce(). Some lines talk about a guard waking up, some about the guard falling asleep, and some about a new guard starting a shift. The problem with that is all of my conditional logic lives inside the body of the callback function for .forEach(), which means if I declare a variable in there (with const or let or even var), the variable will be scoped to that body. .match() takes a regular expression as an argument. I used /#(\d)+/ as the regular expression. That sequence of characters means "a literal '#', then one or more digits - and hold on to the digits for later". The return value of .match() is an array, and the 1st element is the first captured group from the match (the digits I held on to). That's how I get the id out of the line. I use the id as a key in my guards object. If it's the first time I've seen the id, I initialize a value for it as an empty array. That will hold the list of minutes the guard spends asleep.

Once again, a regular expression capture group extracts some digits from a string. This time, I have to feed that value into the Number function, because I will need to treat it as a number in a minute (so to speak). I also initialized the sleepStart variable outside of the callback function so it survives from one loop to the next.

If a line doesn't say a guard's shift begins and it doesn't say a guard falls asleep, it must say the guard wakes up. These things happen in order, so by the time I reach a line that tells me when a guard wakes, I must already have values for currentGuardId and sleepStart. I can use those values, along with the time the guard wakes up, to say which minutes the guard has slept during the latest nap:

  } else if (line.includes('falls')) {

  } else {

wakeTime works exactly the same way sleepStart did. Those two variables together mark the start and end of the interval the guard was asleep. I can use them together to make newMinutes, the array with each minute in that interval. First new Array(wakeTime) makes an array with wakeTime slots in it. (Due to a quirk in JavaScript, I can't do anything useful with that array right off, so I have to fill the empty slots in it by calling .fill(1), which gives me an array of all 1's.) I can convert the array to a sequence of integers by calling .map() and returning the index value of each element in the array. At that point, I'll have an array of 0 to just before wakeTime. For example, if the guard woke up at 12:06, the array will be [0, 1, 2, 3, 4, 5]. I still need to remove the numbers at the front end from before the guard fell asleep, which is what .slice(sleepStart,) does.

Once I have the array with the minutes the guard slept during that nap, I spread them into the array stored at guards[currentGuardId]. Then it is the work of but a moment to find the id of the guard who slept the most. I have to turn the guards object into an array of the object's values, then map those values to their lengths, then get the biggest length. That will tell me the largest number of minutes any guard spent asleep. Then iterate over the keys in guards to find the guard who slept that much:

const times = Object.values(guards).map(guard => guard.length)
const maxTime = Math.max(...times)
const sleepyGuardId = Object.keys(guards).find(
  guardId => guards[guardId].length === maxTime
